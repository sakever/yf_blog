微信AI僚机（WeChat Wingman）——技术方案

# 一、需求背景

内向用户希望在与心仪女生的微信聊天中，获得类似 GalGame（恋爱模拟游戏）的对话体验——即每条对方消息到达后，系统提供三个候选回复选项，用户选择最优的一条发送。核心理念：AI 辅助措辞而非替代聊天，用户保有最终决策权。

两个核心约束：

  1. 不暴露工具：控制端不可出现在微信上（不发给自己微信小号），避免意外暴露
  2. 人格镜像：AI 回复需模仿用户的说话风格，同时叠加高情商、幽默的处理

# 2.1 分阶段规划

阶段 目标 预计工期 技术栈
Phase 1 原型 基础消息收发 + CLI三选一 1-2周 wxauto + LLM API + Python CLI
Phase 2 记忆 Mem0 记忆层，记住对方偏好/话题 1周 Mem0 + ChromaDB
Phase 3 镜像 LoRA微调，让AI更像用户本人 2-4周 LoRA + Qwen微调
Phase 4 面板 移动端/Web控制面板，替代CLI 2-3周 React/Vue + WebSocket

# 三、微信接入层 —— wxauto

# 3.1 选型对比

方案 Stars 原理 封号风险 维护状态 评价
wxauto (cluic/wxauto) ★7,033 Windows UI自动化(UIA) 低-中 活跃 ✅ 推荐选择
WeChatFerry (lich0821) ★6,623 DLL注入/Hook 中-高 活跃 逆向工程，微信频繁封杀
ItChat (littlecodersh) ★26,464 网页版微信协议 极高 停更 ❌ 网页版微信已全面封禁
CowAgent (原chatgpt-on-wechat) ★44,776 上层框架，切换底层 取决于方案 活跃 上层框架，非接入层

最终选型：wxauto

理由：

  • UI自动化而非注入/Hook，微信更新不受直接影响
  • 模拟真人操作节奏，封号风险最低
  • 纯 Python 库，pip install wxauto 即可使用
  • 需 Windows 环境运行（宿主机 Win11 满足条件）

# 3.2 微信版本要求

  • 微信 PC 客户端：3.9.8.x ~ 3.9.11.x（wxauto 测试通过的版本范围）
  • 避免自动更新：建议安装特定版本后关闭微信的自动更新
  • 运行前需先手动登录微信（扫码登录后 wxauto 接管）

# 3.3 防封号策略

策略 说明
操作间隔 收到消息后随机延迟 3-8 秒再回复（模拟真人打字思考）
活跃时段限制 仅在用户设定的时段运行（如 18:00-23:00）
频次限制 单小时最多回复 N 条，超过则停止自动通知
内容多样性 LLM 确保每次回复措辞不同，避免模式化被识别
不操作群聊 仅处理单聊消息，群聊不干涉

# 4.2 Phase 4 升级：Web/App 控制面板

  • 实时推送消息通知（WebSocket）
  • 移动端友好 UI，手机随时选回复
  • 支持快捷自定义输入
  • 聊天历史回看

# 4.3 选项生成策略

LLM 生成三个回复时遵循差异化原则：

选项 风格 适用场景
选项1 共情 + 安抚 对方表达负面情绪
选项2 幽默 + 轻松 对方情绪中性或想活跃气氛
选项3 推进 + 邀约 对话气氛良好时，尝试拉近距离

实际回复会结合上下文动态调整，不机械套用模板。

# 五、LLM 推理层

模型 中文能力 成本(1M token) 延迟 推荐度

推荐 DeepSeek-V3：中文最优、成本极低、延迟适中。

# 5.2 System Prompt 设计

核心结构：

## 角色
你是 [用户名字]，[年龄]，[职业]，[性格特征]

## 说话风格
- 句子简短，不用长难句
- 常用 emoji：[常用表情列表]
- 习惯用语：[口头禅]
- 示例对话风格：[3-5 条真实对话]

## 聊天对象
对方是 [名字]，[关系描述]，[已知偏好]

## 情商规则
1. 先共情再回应——不要直接给解决方案
2. 适当幽默——用轻松的调侃打破沉闷
3. 主动推进——在合适时机引导邀约
4. 记住细节——对方提过的事要在后续对话中自然引用
5. 避免查户口——不要连续提问，穿插分享自己的事

# 七、LoRA 人格微调（Phase 3）

# 7.1 数据准备

  • 收集用户本人微信聊天记录（目标：1000+ 条对话）
  • 格式化为 instruction-following 对：
    {
  "instruction": "对方说：今天好累",
  "output": "啊我也是，今天开了四个会..."
}
    1
    2
    3
    4

# 7.2 微调方案

参数 值
基座模型 Qwen2.5-7B-Instruct
方法 LoRA (r=16, alpha=32)
训练量 1000~3000 条对话
硬件 RTX 4090 24GB / A100 40GB
框架 LLaMA-Factory (简单易用，有 GUI)
时长 ~2-4 小时（单卡 4090）

# 7.3 情商叠加策略

微调让模型"像你"，但你的聊天记录可能本来就"不够高情商"。因此：

最终回复 = 微调模型(你的风格) + System Prompt(情商提升指令)
1

两层叠加：底层是风格，上层是策略。就像你本人 + 一个高情商的"内心旁白"。

# 九、风险与应对

风险 等级 应对
微信封号 🔴 高 wxauto 降低风险；建议用非主力微信号；控制操作频率
LLM 回复不当 🟡 中 用户最终决策，不当回复不发；增加敏感词过滤
延迟过长 🟢 低 DeepSeek API 响应快；支持异步处理
隐私泄露 🟡 中 本地部署 Mem0 + ChromaDB；聊天数据不离开本机
伦理争议 🟡 中 明确"辅助措辞"定位，用户参与每一条回复的决策
